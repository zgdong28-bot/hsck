## 布局方案

### 主题切换按钮位置

放在 header-top 右侧，与语言切换按钮并列。三个页面都有 `header-top` 区域（flex 布局，两端对齐），右侧当前只有 `lang-toggle`。

```
┌─────────────────────────────────────────┐
│ [header-label]                          │
│ Page Title          [🌙/☀️] [EN/中文]   │
│ [nav links...]                          │
└─────────────────────────────────────────┘
```

按钮样式复用 `.lang-toggle` 的设计语言（monospace、小号、边框按钮），保持 header 视觉一致性。

### 变体 A：极简双态切换（推荐）

**核心思路**：暗色保持赛博朋克全部效果，亮色走干净的技术文档风——去掉扫描线和网格，保留绿色强调色。

**亮色色板**：
```css
[data-theme="light"] {
  --bg: #f4f6f9;
  --surface: #ffffff;
  --surface-2: #eef1f5;
  --border: #d4dae3;
  --border-glow: #b8c2d0;
  --green: #00b37e;        /* 略深，白底上对比度更好 */
  --green-dim: #00b37e1a;
  --green-glow: #00b37e33;
  --amber: #d08a20;
  --amber-dim: #d08a201a;
  --red: #e0304d;
  --red-dim: #e0304d1a;
  --blue: #3a8ee6;
  --blue-dim: #3a8ee61a;
  --text: #2c3e50;
  --text-dim: #8896a7;
  --text-bright: #1a202c;
}
```

**背景处理**：
- `body::after`（扫描线）：`opacity: 0` 隐藏
- `body::before`（网格+辉光）：`opacity: 0` 隐藏
- 亮色下 body 纯色背景 `--bg`，干净清爽

**优势**：
- 改动量最小，亮色只做变量覆盖 + 隐藏纹理
- 两套主题视觉差异明确，不会出现"暗不暗亮不亮"的尴尬
- 绿色强调色在两个主题下都保持品牌识别度

**劣势**：
- 亮色主题没有任何装饰纹理，可能略显朴素

### 变体 B：亮色保留微纹理

**核心思路**：亮色下保留极淡的网格背景（opacity 降到 0.03），带一点质感。

**背景处理**：
```css
[data-theme="light"] body::after { opacity: 0; }           /* 扫描线完全去掉 */
[data-theme="light"] body::before {
  background:
    linear-gradient(rgba(0,0,0,0.02) 1px, transparent 1px),
    linear-gradient(90deg, rgba(0,0,0,0.02) 1px, transparent 1px);
  background-size: 60px 60px;
}
```

**优势**：亮色下有微妙的"工程图纸"质感，与赛博朋克暗色在精神上有关联
**劣势**：需要额外调试浅色网格的视觉效果，可能在某些屏幕上看起来脏

### JS 主题逻辑（加入 shared-ui.ts）

```javascript
function getTheme() {
  return localStorage.getItem('theme') || 'dark';
}

function applyTheme(theme) {
  document.documentElement.setAttribute('data-theme', theme);
  const btn = document.getElementById('themeToggle');
  if (btn) btn.textContent = theme === 'dark' ? '☀️' : '🌙';
}

function toggleTheme() {
  const next = getTheme() === 'dark' ? 'light' : 'dark';
  localStorage.setItem('theme', next);
  applyTheme(next);
}
```

页面加载时在 `<script>` 开头立即调用 `applyTheme(getTheme())`，防止闪烁。

### 防闪烁策略

在 `<head>` 中、CSS 之后立即执行一段内联脚本：

```html
<script>
  (function(){
    var t = localStorage.getItem('theme') || 'dark';
    document.documentElement.setAttribute('data-theme', t);
  })();
</script>
```

这段代码在 body 渲染前就设好 `data-theme`，配合 `body { opacity: 0 }`（已有的加载遮罩），实现零闪烁切换。

## 风险点
- **辉光 box-shadow 在亮色下可能过于显眼**：暗色下 `box-shadow: 0 0 20px var(--green-dim)` 是微妙的辉光效果，亮色下因为 `--green-dim` 变为 `#00b37e1a`（低透明度），效果会自然减弱，但需要实测确认
- **第三方内容（如 toast）**：toast 使用 `var(--green)` 背景 + 白色文字，亮色下绿色变深后白色文字对比度需确认
- **login overlay**：使用 `var(--bg)` 作为全屏背景，亮色下会变为浅灰色，确认视觉效果

## 验证方式
- 本地 `npm run dev:cf` 启动后访问三个页面（admin、dashboard、config-editor）
- 点击主题切换按钮，确认暗/亮切换即时生效
- 刷新页面，确认主题选择通过 localStorage 持久化，无闪烁
- 检查所有按钮（primary、danger、secondary）在亮色下的对比度和可读性
- 检查 toast 提示在两个主题下的可见性
- 移动端视口下确认主题按钮可点击且不溢出
