# 不想用云服务？TVBox 源聚合现在支持局域网部署了

上一篇教了大家把 TVBox 源聚合服务部署到 Cloudflare——免费、省心、全球可达。

但后台陆续收到不少反馈：

> "我家 NAS 闲着也是闲着，能不能跑在本地？"
> "公司网络访问不了 workers.dev，本地搭一个多好"
> "我就想在家里局域网用，不需要外网访问"

想了想，确实。**不是每个人都需要一个"在线服务"**——很多人的场景就是家里几台电视、手机，全在一个局域网里。放在 NAS 上或者家里一台旧电脑上跑，反而更快、更稳、更可控。

所以这次做了一次大更新：**除了原来的 Cloudflare 方案，现在支持 Docker 部署和本地一键启动了。**

---

## 先看效果

不管哪种部署方式，最终你得到的东西都一样：

- **一个局域网地址**（比如 `http://192.168.1.100:5678/`），TVBox 填这个地址就能用
- **一个管理后台**（`/admin`），网页上管理你的源
- **一个监控页面**（`/status`），随时看聚合状态
- **每天自动更新**，定时抓取 → 测速 → 去重 → 合并

和云端版本功能完全一致，只是跑在你自己的设备上。

---

## 三种部署方式怎么选

| 方式 | 适合谁 | 难度 | 一句话 |
|:---|:---|:---|:---|
| **Docker** | 有 NAS 或 Linux 服务器 | 最简单 | 一行命令搞定 |
| **一键脚本** | 有 Mac/Windows 电脑 | 简单 | 双击文件就能跑 |
| **Cloudflare Worker** | 想要外网访问 | 中等 | 免费云端，参考上一篇 |

**推荐**：有 NAS 的直接上 Docker，没有的用一键脚本。Cloudflare 方案适合需要从外网访问的场景。

---

## 方式一：Docker 部署（NAS / Linux 服务器）

最省心的方案。群晖、威联通、Unraid、Linux 服务器都行。

### Step 1 · 克隆仓库

```bash
git clone https://gitee.com/tengxiaobao/tvbox-source-aggregator.git
cd tvbox-source-aggregator
```

### Step 2 · 配置密码

```bash
cp .env.example .env
```

编辑 `.env`，把管理密码改掉：

```
ADMIN_TOKEN=你的管理密码
PORT=5678
```

就这两行，其他都可以不动。

### Step 3 · 启动

```bash
docker compose up -d
```

没了。打开浏览器访问 `http://你的NAS_IP:5678/admin`，就能看到管理后台了。

### 群晖用户补充

群晖的 Docker 通过 Container Manager 管理。如果你习惯命令行操作，SSH 登录后步骤完全一样。如果 `docker` 命令找不到，试试：

```bash
export PATH="/volume1/@appstore/ContainerManager/usr/bin:$PATH"
```

数据文件默认保存在项目目录下的 `data/` 文件夹里。用 SQLite 存储，重启容器数据不会丢。

## 方式二：一键脚本（Mac / Windows）

不想折腾 Docker？双击一个文件就能启动服务。

### Step 1 · 克隆仓库

```bash
git clone https://gitee.com/tengxiaobao/tvbox-source-aggregator.git
```

或者直接在 Gitee 页面下载 ZIP 解压。

### Step 2 · 双击启动

**Mac 用户**：打开项目文件夹，双击 `start.command`

**Windows 用户**：打开项目文件夹，双击 `start.bat`

首次启动会自动完成以下所有检查：

```
✓ 检查 Node.js（没有会提示安装）
✓ 安装项目依赖
✓ 创建数据目录
✓ 引导你设置管理密码
✓ 检查端口是否冲突
✓ 编译并启动服务
```

启动成功后，终端会显示你的局域网地址：

```
  TVBox Source Aggregator
  > Local:   http://localhost:5678/
  > Network: http://192.168.1.100:5678/
  > Admin:   http://192.168.1.100:5678/admin

  TVBox 填入地址: http://192.168.1.100:5678/
```

**Network 那行就是你要在 TVBox 里填的地址。**

### Node.js 没装怎么办

- **Mac**：脚本会尝试通过 Homebrew 自动安装；如果没有 Homebrew，会下载官方安装包
- **Windows**：脚本会自动下载安装 Node.js LTS 版本

实在不行，手动去 [nodejs.org](https://nodejs.org) 下载 LTS 版本装一下，然后重新双击启动文件。

## 添加源

不管哪种部署方式，添加源的步骤都一样：

1. 浏览器打开 `http://你的地址:5678/admin`
2. 输入你设置的管理密码
3. 粘贴 TVBox 配置 JSON 地址，点 Add
4. 点 Refresh 触发一次聚合
5. 等十几秒，访问首页确认配置已生成

**源从哪来？** 你手上现有的 TVBox 接口地址——就是之前在 TVBox 里填的那些 JSON 地址——直接加进来。多加几个，系统自动去重合并。

不知道源的话：随便找个 AI 工具问"帮我整理一份 TVBox 的源"，加进来就行。

---

## TVBox 里填地址

把你的聚合地址填进 TVBox 的接口设置：

```
http://你的局域网IP:5678/
```

保存，返回首页，内容就出来了。

---

## 可选：开启测速

默认会把所有源合并。如果想自动过滤掉响应慢的源：

1. 注册 [zbape.com](https://www.zbape.com)，获取免费 API Key
2. 在 `.env` 中添加：

```
ZBAPE_API_KEY=你的API_KEY
```

Docker 用户改完 `.env` 后 `docker compose restart` 生效。

---

## 可选：后台常驻运行

一键脚本默认是前台运行的——关了终端窗口服务就停了。想让它一直跑：

```bash
# 方法一：pm2（推荐）
npm install -g pm2
pm2 start scripts/start.js --name tvbox

# 方法二：nohup
nohup node scripts/start.js > tvbox.log 2>&1 &
```

Docker 用户不用管这个，容器本身就是后台运行的，而且会自动重启。

## 常见问题

| 问题 | 解决 |
|:---|:---|
| **Docker 启动报错** | 确认 `.env` 文件存在且 `ADMIN_TOKEN` 已修改 |
| **双击 start.command 没反应** | 右键 → 打开方式 → 终端；或者在终端执行 `chmod +x start.command` 后重试 |
| **端口被占用** | 修改 `.env` 中的 `PORT` 换一个端口 |
| **TVBox 加载空白** | 确认地址末尾有 `/`；确认已添加源并点了刷新 |
| **局域网其他设备访问不了** | 检查防火墙设置；确认用的是 Network 地址而不是 localhost |
| **群晖找不到 docker 命令** | `export PATH="/volume1/@appstore/ContainerManager/usr/bin:$PATH"` |
| **想更新到新版本** | `git pull && docker compose up -d --build`（Docker）；`git pull` 后重新双击启动（脚本） |

## 和上一篇的区别

上一篇的 Cloudflare Worker 方案**依然可用**，适合需要从外网访问的场景。

这次新增的 Docker 和一键脚本方案，核心变化是：

- **存储**：从 Cloudflare KV 换成了 SQLite（Docker/本地）——数据在你自己手里
- **运行时**：从 Cloudflare Worker 换成了标准 Node.js——想跑在哪台机器跑哪台
- **启动**：不需要命令行操作——双击文件或者一行 Docker 命令

三种方式共享同一套核心代码（源聚合、去重、测速逻辑完全一致），只是运行环境不同。

## 一句话总结

> **有 NAS → Docker 一行命令。没 NAS → 双击启动文件。想要外网 → Cloudflare。**
> 
> 不用再纠结部署方式了，选一个最顺手的就行。
