## 背景

TVBox-Pro 客户端加载源站图片（海报、封面）时直连源站，速度慢。
聚合器端已部署 CF Worker 图片代理端点 `/img/*`，需要客户端配合使用。

## 原理

在配置 JSON 根级别新增 `pic` 字段，TVBox 加载图片时自动拼接代理前缀：

```
原始图片 URL:  https://img.example.com/poster.jpg
pic 字段值:    https://tvbox.rio.edu.kg/img/
拼接后:        https://tvbox.rio.edu.kg/img/https://img.example.com/poster.jpg
               ↓
         CF Worker fetch 原图 → 边缘缓存 7 天 → 返回客户端
```

### 2. ImgUtil.java

**文件**: `app/src/main/java/com/github/tvbox/osc/util/ImgUtil.java`

#### 2.1 修改 getUrl() 方法（第 191 行之后）

在 `checkReplaceProxy` 之后、header 解析之前插入：

```java
// 现有代码
private static Object getUrl(String url) {
    if (url.startsWith("data:")) return url;
    url = DefaultConfig.checkReplaceProxy(url);

    // ▼ 新增：pic 图片代理前缀
    String picProxy = ApiConfig.get().getPicProxy();
    if (!TextUtils.isEmpty(picProxy) && (url.startsWith("http://") || url.startsWith("https://"))) {
        url = picProxy + url;
    }

    // 以下为现有代码，不变
    String header = null;
    String referer = null;
    // ...
```

## 注意事项

1. **`proxy://` 协议不受影响** — pic 前缀只对 `http://`/`https://` 开头的 URL 生效，`proxy://` 走原有逻辑
2. **`@Header` 参数兼容** — pic 前缀加在完整 URL（含 @Header 后缀）之前，ImgUtil 后续解析 `@` 时会正确拆分
3. **空值安全** — `pic` 字段不存在或为空时，`picProxy` 默认为 `""`，不影响现有行为
4. **wallpaper 也会走代理** — wallpaper 如果通过 ImgUtil 加载，同样会走 pic 代理，这是预期行为

## 验证方式

1. 在配置 JSON 中添加 `"pic": "https://tvbox.rio.edu.kg/img/"`
2. 打开 TVBox，进入任意 JAR 源，浏览影视内容
3. 检查图片是否正常加载（Logcat 过滤 `ImgUtil` 或 `Glide`）
4. 在 CF Dashboard → Workers → Analytics 中确认 `/img/*` 请求量
